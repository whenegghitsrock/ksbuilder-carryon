# [[ .Name ]] — frontend scaffold

In **ksbuilder** this tree lives at `pkg/extension/templates/frontend/extensions/scaffold/` and is copied to **`frontend/extensions/<your-extension-name>/`** when you run create/extend.

This directory is generated by **ksbuilder** and lines up with the **Hello World** extension layout used in the KubeSphere docs and upstream samples.

**Note:** `ksbuilder create` copies **`src/`** only (including **`src/console-shell.js`**, the SystemJS shell source). **`dist/index.js`** and **`dist/assets/`** are produced by **`npm run build:shell`** / **`npm run build:assets`** or inside **`docker build`** (see below).

## How the pieces connect (cluster install)

```text
Console loads JSBundle → dist/index.js (System.register)
       → iframe src = /proxy/<ext>/index.html
              → nginx serves index.html + assets/bundle.js
                     → assets/bundle.js is built in Docker (or locally via npm) from src/iframe.jsx → App.jsx
```

1. **`dist/index.js`** — Built from **`src/console-shell.js`** (SystemJS extension entry: menu, route, iframe → **`/proxy/<extension-name>/index.html`**). Created by **`npm run build:shell`** or the Docker Node stage.
2. **`index.html` + `assets/bundle.js`** — The page **inside** the iframe. The **Dockerfile** Node stage runs **`npm run build:assets`** and **`npm run build:shell`**, then nginx copies artifacts into the image.
3. **`src/index.js` / `routes/` / `locales/`** — Used only when you develop **inside** a KubeSphere Console repo (`yarn dev`); they call **`globals.context.registerExtension`**. They are **not** used by the Node stage for the nginx image.

## Layout in generated extensions

ksbuilder keeps the directory name **`frontend/`** at the extension root. Inside it:

- **Workspace** (mirrors [extension-samples `extensions-frontend`](https://github.com/kubesphere/extension-samples/tree/master/extensions-frontend) root): `package.json`, `yarn.lock`, `babel.config.js`, `configs/`, etc.
- **This extension:** `extensions/[[ .Name ]]/` — Docker/nginx image build (`Makefile`, `Dockerfile`, **`src/console-shell.js`**, …); **`dist/`** appears after build.

## Build commands

| Goal | Command |
|------|---------|
| Docker image (recommended; builds iframe bundle inside Docker) | `cd frontend/extensions/[[ .Name ]] && make build` |
| Local-only **`dist/assets/bundle.js`** (optional, for inspection) | `cd frontend/extensions/[[ .Name ]] && make build-assets` |
| From extension root | `make build-frontend` |
| Install workspace deps (Console / `ksc` development) | `cd frontend && yarn install` |

## 1. Cluster image (Helm / JSBundle)

- **SystemJS entry:** `dist/index.js` (generated from `src/console-shell.js` during image build)
- **Iframe app:** built during **`docker build`** from **`src/`** + **`package.json`** / **`package-lock.json`**
- **Build/push:** extension root **`Makefile`** → **`make -C frontend`** (wrapper forwards into **`extensions/[[ .Name ]]`**)

## 2. Local KubeSphere Console (Hello World tutorial)

The **`src/`** tree (except **`iframe.jsx`**) matches the official Console extension sample:

- Upstream: [extension-samples — hello-world](https://github.com/kubesphere/extension-samples/tree/master/extensions-frontend/extensions/hello-world)
- Guide (zh): [创建 Hello World 扩展组件](https://dev-guide.kubesphere.io/extension-dev-guide/zh/quickstart/hello-world-extension/)
- Guide (en): [Hello World quickstart](https://dev-guide.kubesphere.io/extension-dev-guide/en/quickstart/hello-world-extension/)

Flow: open the generated **`frontend/`** as the extensions workspace → add **`configs/extensions/**`** as in the dev guide → **`yarn dev`**.

## File layout (reference)

```text
frontend/                          # workspace root (yarn; same role as extension-samples/extensions-frontend)
├── package.json
├── yarn.lock
├── babel.config.js
├── configs/
│   ├── console.config.js
│   └── extensions/               # local dev extension configs (often empty at create time)
└── extensions/
    └── [[ .Name ]]/
        ├── README.md
        ├── package.json
        ├── package-lock.json
        ├── Dockerfile
        ├── Makefile
        ├── index.html
        ├── dist/                     # after build: index.js, assets/bundle.js
        └── src/
            ├── console-shell.js     # SystemJS shell source → dist/index.js
            ├── iframe.jsx
            ├── App.jsx
            ├── index.js
            ├── locales/
            └── routes/
```

## zh 摘要

- 扩展根目录下始终为 **`frontend/`**，不会改名为 **`extensions-frontend`**。
- **`ksbuilder create`** 只带 **`src/`**（含 **`console-shell.js`**），**不包含** 预生成的 **`dist/`**（由 build / **`build:shell`** 生成 **`dist/index.js`**，由 **`build:assets`** 生成 **`dist/assets/`**）。
- **镜像构建**：在 **`frontend/extensions/[[ .Name ]]`** 下执行 **`make build`**（或扩展根目录 **`make build-frontend`**）。
- **Console 本地开发**：在 **`frontend/`** 下 **`yarn install`**，并按文档维护 **`configs/extensions/`**。
