# [[ .Name ]] — frontend scaffold

This directory is generated by **ksbuilder** and lines up with the **Hello World** extension layout used in the KubeSphere docs and upstream samples.

**Note:** `ksbuilder create` only copies **`dist/index.js`** (SystemJS shell) and **`src/`** sources. It does **not** copy **`dist/assets/`** — that directory is produced when you build (see below).

## How the pieces connect (cluster install)

```text
Console loads JSBundle → dist/index.js (System.register)
       → iframe src = /proxy/<ext>/index.html
              → nginx serves index.html + assets/bundle.js
                     → assets/bundle.js is built in Docker (or locally via npm) from src/iframe.jsx → App.jsx
```

1. **`dist/index.js`** — Shell extension entry: menu, route, and an iframe pointing at **`/proxy/<extension-name>/index.html`** (same origin as Console; `ReverseProxy` strips `/proxy/<name>` and forwards to the frontend Service).
2. **`index.html` + `assets/bundle.js`** — The page **inside** the iframe. The **Dockerfile** uses a **Node build stage** to run **`npm run build:assets`** and place **`bundle.js`** under **`/usr/share/nginx/html/assets/`**. Alternatively, run **`npm install && npm run build:assets`** locally to create **`dist/assets/bundle.js`** for debugging.
3. **`src/index.js` / `routes/` / `locales/`** — Used only when you develop **inside** a KubeSphere Console repo (`yarn dev`); they call **`globals.context.registerExtension`**. They are **not** used by the Node stage for the nginx image.

## Build commands

| Goal | Command |
|------|---------|
| Docker image (recommended; builds iframe bundle inside Docker) | `cd frontend && make build` |
| Local-only **`dist/assets/bundle.js`** (optional, for inspection) | `cd frontend && make build-assets` |
| From extension root | `make build-frontend` |

## 1. Cluster image (Helm / JSBundle)

- **SystemJS entry:** `dist/index.js` (from create template)
- **Iframe app:** built during **`docker build`** from **`src/`** + **`package.json`** / **`package-lock.json`**
- **Build/push:** extension root **`Makefile`** / **`make -C frontend`**

## 2. Local KubeSphere Console (Hello World tutorial)

The **`src/`** tree (except **`iframe.jsx`**) matches the official Console extension sample:

- Upstream: [extension-samples — hello-world](https://github.com/kubesphere/extension-samples/tree/master/extensions-frontend/extensions/hello-world)
- Guide (zh): [创建 Hello World 扩展组件](https://dev-guide.kubesphere.io/extension-dev-guide/zh/quickstart/hello-world-extension/)
- Guide (en): [Hello World quickstart](https://dev-guide.kubesphere.io/extension-dev-guide/en/quickstart/hello-world-extension/)

Flow: `create-ks-project` → `yarn create:ext` → copy or compare **`src/`** (and **`package.json`**) under **`extensions/<name>/`** → **`local_config.yaml`** → **`yarn dev`**.

## File layout (reference)

```text
frontend/
├── README.md
├── package.json          # npm scripts; build:assets writes dist/assets/bundle.js
├── package-lock.json     # used by npm ci in Dockerfile
├── Dockerfile            # multi-stage: Node builds assets → nginx serves static files
├── Makefile
├── index.html
├── dist/
│   └── index.js          # from ksbuilder create (SystemJS entry for Console host)
│   # dist/assets/ is not created by create; appears after npm run build:assets or docker build
└── src/
    ├── iframe.jsx        # cluster iframe entry (mount App)
    ├── App.jsx
    ├── index.js          # Console-only registerExtension
    ├── locales/
    └── routes/
```

## zh 摘要

- **`ksbuilder create`** 只带 **`dist/index.js`** 和 **`src/`**，**不包含** 预生成的 **`dist/assets/`**。
- **镜像构建**：**Dockerfile** 里用 Node 阶段执行 **`npm run build:assets`**，再在 nginx 阶段拷贝产物；平时 **`make build`** 即可。
- **本地调试**：需要看落地文件时可在 **`frontend/`** 下执行 **`make build-assets`**，会生成 **`dist/assets/bundle.js`**。
- **Console 本地开发**：仍用 **`src/index.js`** 等与官方 Hello World 一致的 **`registerExtension`** 流程（见 dev-guide）。
